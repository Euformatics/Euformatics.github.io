Using the omnomicsQ command line application Docker image

The omnomicsQ Docker image is an alternative way to obtain the omnomicsQ command-line application. Benefits of using the omnomicsQ Docker image include that you donâ€™t need to install or configure the application on your machine, as long as you have a working Docker installation (including Docker Desktop if you are using Windows or Mac). Since there is nothing to install, upgrading to a newer version is just a matter of referencing a newer version number on your command line.

Obtaining the omnomicsQ Docker image

The Docker image is available through the Docker Hub at euformatics/omnomicsq_cli. Check the linked page for the latest available version. The latest version at the time of writing is 1.1.6. You can pull it to your machine with the following command:

docker pull euformatics/omnomicsq_cli:1.1.6

Running the omnomicsQ Docker image from the command line

The command line application connects to the omnomicsQ server at q.omnomics.com over HTTPS. A username and a password are used to authenticate. For testing and occasional usage, you can use the same username and password that you use to log in to the web application. For configuring the command line application as a permanent part of your pipeline, please contact Euformatics support at support@euformatics.com to obtain another password that is separate from the password you use to interactively use the application.

Username and password are specified via the oq_org and oq_license environment variables. You need to specify these environment variables with the -e command line switch when you run the command line application.

In the Bash shell on Linux/Mac, you can set the environment variables with the export command. Replace <YOUR_USERNAME> and <YOUR_PASSWORD> with your actual username and password:

export oq_org=<YOUR_USERNAME>
export oq_license=<YOUR_PASSWORD>

If you are on Windows (using Docker Desktop), in Command Prompt or PowerShell you will need to use the set command instead:

set oq_org=<YOUR_USERNAME>
set oq_license=<YOUR_PASSWORD>

To ensure that your username and password are correct, you can now try using docker run to execute the omnomicsq_cli list_devices command for listing your configured devices from the server:

docker run -i -e oq_org -e oq_license euformatics/omnomicsq_cli:1.1.6 /opt/omnomicsq/omnomicsq_cli list_devices

If everything was successful, you will see a list of your devices.

Getting started

You can issue the following command to see the command line help:

docker run -i -e oq_org -e oq_license euformatics/omnomicsq_cli:1.1.6 /opt/omnomicsq/omnomicsq_cli --help

The quick start guide is included as a README file with the docker image, and it contains example commands for common tasks like analyzing FASTQ, BAM and VCF files. You can view it with the following command:

docker run -i -e oq_org -e oq_license euformatics/omnomicsq_cli:1.1.6 cat /opt/omnomicsq/README.md
